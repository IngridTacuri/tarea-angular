ROUTING

Definición

Es un mecanismo en las aplicaciones web o móviles modernas, ya sean aplicaciones de una sola página, aplicaciones web progresivas o aplicaciones móviles.


Configuración

Crear los componentes necesarios respecto a la arquitectura de la navegación del aplicativo.

Para crear el componente, se ejecuta la siguiente línea de la terminal: ng generate component nombre-component

Abrir app-routing.module.ts e importe los componentes para los que se deben configurar para routing: import nombre-componentComponent from ./nombre-component/nombre-component.component;

Insertar la ruta dentro de const routes: Routes = {{'{'}}{{'['}} path: 'nombre-component', component: nombre-componentComponent }];

DATA-BINDING

Conceptos

Definición

Es la forma que tiene Angular para permitirnos mostrar contenido dinámico en lugar de estático (en inglés, hardcoded). Podríamos traducir el databinding como "comunicación" entre nuestro código HTML (archivo .html) y nuestra lógica de programación (archivo .ts).


Tipos

String interpolation: Permite mostrar información en el template desde el typescript. Se representa por una doble llave abierta y su correspondiente cierre.


Property binding: Es utilizado para enviar informacion de los datos a la vista, en concreto con propiedades de componentes, elementos HTML y directivas.


Event binding: Se usa para manejar la reacción a eventos generados por el usuario.

PadreComponent - HijoComponent

¿Cómo se identifican?

Consiste en usar la etiqueta @Input de Angular. Ésta etiqueta se pone en el componente hijo para indicar que esa variable proviene desde fuera del componente, es decir desde el componente padre usamos el selector del hijo para incluirlo en el html y le pasamos el valor que queremos pasar al hijo.
